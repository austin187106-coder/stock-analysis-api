## 一、測試過程中發現並修正的問題

在撰寫測試的過程中，實際比對函式庫行為後，發現原架構中有兩處會讓計算結果偏離預期的缺陷，已一併修正：

### 🔴 缺陷一：布林通道標準差參數未生效（已修正）

`indicators_module.py` 原先呼叫 `ta.bbands(..., std=self.bb_std)`。實際比對 `pandas_ta` 函式庫的函式簽章後發現，目前安裝的版本（0.4.71b0）並沒有 `std` 這個參數，正確名稱是 `lower_std` / `upper_std`。由於 Python 會把不存在的關鍵字參數吃進 `**kwargs` 而不報錯，這個錯誤**不會丟出任何例外**，只會讓 `bollinger_std` 設定值被靜默忽略，永遠套用函式庫內建的預設值（2.0）。

也就是說：即使在 `config.py` 把 `bollinger_std` 改成 1.5 或 2.5，計算出來的通道寬度也完全不會改變。

**修正方式**：改為 `ta.bbands(length=..., lower_std=self.bb_std, upper_std=self.bb_std)`。
**驗證測試**：`test_bollinger_std_parameter_scales_bandwidth`（修正前此測試會失敗，比值恆為1.0；修正後比值正確反映為2.0）。

### 🔴 缺陷二：資料筆數不足時會直接崩潰（已修正）

當輸入資料的筆數小於指標所需的最小週期（例如只有5筆資料卻要計算20MA或MACD(12,26,9)）時，`pandas_ta` 會回傳 `None` 而不是含NaN的Series。原程式碼直接對回傳值取 `.columns`，導致拋出 `AttributeError: 'NoneType' object has no attribute 'columns'`，使整個 API 請求中斷。

**修正方式**：在 `add_moving_average`／`add_macd`／`add_bollinger_bands` 中加入 `None` 檢查，資料不足時改以 `NaN` 填入對應欄位，讓後續流程（包含 Logic 模組）能依現有的NaN安全機制正常運作，而不是讓整個請求例外中斷。
**驗證測試**：`test_insufficient_rows_returns_nan_instead_of_crashing`、`test_empty_dataframe_does_not_raise`。

> **附帶說明**：這也解釋了為什麼 `data_module.py` 預設抓取兩年資料——以日線推算月線時，兩年大約對應24根月線，剛好足夠計算月線20MA；若改成抓取一年資料，月線20MA會因資料不足而變成NaN，第一層濾網會安全地判定為不通過，而不會出錯。實際以400天模擬資料跑過完整三層濾網流程後，也確認了這個保護機制在真實情境下能正常運作。

## 二、邏輯覆蓋測試（對應需求 1）

依需求模擬三種情境，確認最終訊號是否符合預期：

| 情境 | 月線 | 週線 | 日線 | 預期 is_buy_signal | 實際結果 |
|---|---|---|---|---|---|
| (a) 完全符合買進條件 | 通過（站上月線20MA、MACD柱狀體轉正） | 通過（站上週線20MA與布林中軌） | 通過（站上布林中軌、未觸及上緣、較前日上漲） | `True` | ✅ `True` |
| (b) 週線向上但日線未突破 | 通過 | 通過（站上均線，趨勢向上） | **不通過**（當日收盤跌破布林中軌且較前日下跌） | `False` | ✅ `False`，`summary` 明確標示「未通過：日線濾網」 |
| (c) 布林通道未開口（緊縮） | 通過 | 通過 | 見下方說明 | — | ⚠️ 見下方說明 |

**情境 (c) 的重要發現**：目前 `logic_module.py` 的第三層只檢查「站上中軌」「未超過上緣」「較前日上漲」，**並未檢查通道寬度（頻寬）本身**。測試刻意建構通道寬度僅為中軌價格 1%（遠低於正常波動）、且當日僅上漲 0.2% 的緊縮情境，結果第三層仍判定為「通過」，最終 `is_buy_signal` 仍為 `True`。

這不是程式錯誤，而是**目前交易邏輯尚未涵蓋的情境**——嚴格來說，布林通道緊縮代表市場處於低波動的觀望期，此時微幅價格波動並不足以構成「真突破」的訊號，理論上應該被濾掉，但目前規則並未要求頻寬達到一定程度才算數。

**建議**：可在 `check_daily_layer` 中加入頻寬濾網，例如：
```python
bandwidth_pct = (latest["bb_upper"] - latest["bb_lower"]) / latest["bb_mid"]
band_expanded = bandwidth_pct > 0.04   # 門檻可依回測結果調整
```
是否要加入這個條件屬於交易策略決策，我先把測試與發現列出來，這部分的程式碼我沒有自行更動，需要的話我可以幫你加上去。

**關於 NaN 在 Logic 模組中的處理機制，有一個值得留意的細節**：`latest.get(ma_col, float("inf"))` 這種寫法，預設值 `float("inf")` 只有在「欄位完全不存在」時才會生效；如果欄位存在但數值是 `NaN`，`.get()` 會回傳 `NaN` 本身。但因為 `收盤價 > NaN` 在 Python／NumPy 中恆為 `False`，所以即使預設值機制沒有被觸發，比較結果仍然會安全地判定為「不通過」，不會誤判為買進訊號，也不會拋出例外。這是個正確但不直觀的行為，已寫成測試並記錄下來，避免未來修改程式碼時不小心破壞這個保護機制。

## 四、參數正確性測試（對應需求 3）

實際以隨機模擬的60筆收盤價，逐一比對程式輸出與標準公式的手動計算結果（誤差皆在 float64 浮點精度範圍內，約 1e-9 ~ 1e-14）：

| 指標 | 標準定義 | 驗證方式 | 結果 |
|---|---|---|---|
| 移動平均線（MA） | N期簡單移動平均 | 與 `close.rolling(N).mean()` 比對 | ✅ 完全吻合 |
| 布林通道中軌 | 等於20期SMA | 與 `rolling(20).mean()` 比對 | ✅ 完全吻合 |
| 布林通道上下緣 | 中軌 ± 2 × 20期標準差 | 與 `rolling(20).std(ddof=1)` 比對 | ✅ 完全吻合（確認採用樣本標準差，非母體標準差） |
| 布林通道對稱性 | 上緣到中軌 = 中軌到下緣 | 直接比對兩段距離 | ✅ 完全對稱 |
| 布林通道標準差倍數參數 | 倍數放大應等比例放大通道寬度 | 比對 std=1 與 std=2 的通道寬度比值 | ✅ 比值=2.0（修正缺陷一後） |
| MACD 線 | EMA(12) − EMA(26) | 與函式庫自身的 `ta.ema()` 交叉比對 | ✅ 完全吻合 |
| MACD 柱狀體 | MACD線 − 訊號線（代數恆等式） | 直接驗證等式關係 | ✅ 完全吻合 |

**補充發現**：`pandas_ta` 的 EMA 採用「TA-Lib 風格」的SMA種子初始化（先用前N筆資料的簡單平均數作為起點，再接續以指數平滑遞迴計算），而不是單純從第一筆資料開始遞迴。這與多數看盤軟體（如 TradingView、TA-Lib）的慣例一致，數值上與業界標準定義吻合，但與最單純的 `Series.ewm(adjust=False)` 寫法不會完全相同，這點在測試中已透過函式庫自身的 `ema()` 函式交叉驗證，避免誤判為缺陷。
